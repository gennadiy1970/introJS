Переменные: объявление let/const, имена переменных

Блок

Блок используется для группировки нуля и более инструкций. Блок отделяется парой фигурных скобок. 


{};
if (true) { 
    console.log('внутри блока')
};

Литералы

Литералы используются для представления значений в JavaScript. Они являются фиксированными значениями, а не переменными.

Переменные

const: область видимости ограничена блоком. Значение не перезаписывается. Не всплывает.

let: область видимости ограничена блоком. Значение перезаписывается. Не всплывает.

var: область видимости везде, кроме блока function. Значение перезаписывается. Всплывает.

Переменная без объявления становится глобальной, т.е. доступной везде. В strict mode это вызовет срогое предупреждение. Это крайне нежелательная практика

Названия

Имена переменных называются identifiers и должны соответствовать определенным правилам.

Начинаются с букв или символов $ и _. Далее можно ставить цифры и буквы в разных регистрах. Можно, но не нужно ставить символы Unicode

Присваивание значений

Переменная, объявленная через var или let без присвоения начального значения, имеет значение undefined.

При попытке доступа к необъявленной переменной или переменной до её объявления будет выброшено исключение ReferenceError:​


var a;
console.log("The value of a is " + a); 
Значение переменной a undefined

console.log("The value of b is " + b); 
Uncaught ReferenceError: 
    b не определена

console.log("The value of c is " + c); 
Значение переменной 
    c undefined
var c;

console.log("The value of x is " + x); 
Uncaught ReferenceError: 
    x не определена
let x;

Поднятие переменных / hoisting

Можно сослаться на переменную, которая объявляется позже, и не получить при этом исключения.

Из-за поднятия переменных, все операторы var в функции следует размещать настолько близко к началу функции, насколько это возможно.

let (const) не будет подниматься вверх блока. Однако, ссылки на переменную в блоке до объявления переменной вызовут ReferenceError
